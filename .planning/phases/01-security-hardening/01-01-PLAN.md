Two targeted edits to `artifacts/api-server/src/routes/events.ts`:

**Fix 1 — Public event GET (line ~406): filter to published only**

Change the WHERE clause from:
```typescript
const [event] = await db.select().from(eventsTable).where(eq(eventsTable.slug, slug));
```
To:
```typescript
const [event] = await db.select().from(eventsTable)
  .where(and(eq(eventsTable.slug, slug), eq(eventsTable.status, "published")));
```
This means a 404 is returned for draft, cancelled, or any non-published event. The `and` import is already present at the top of the file.

Also remove `status` from the public response object — callers of a published-only endpoint don't need it (it's always "published"), and exposing the status field could hint at the filter.

**Fix 2 — Public RSVP POST (line ~454): look up by rsvpToken, not integer ID**

Replace the `guestToken` handling block:
```typescript
// OLD — integer ID lookup (insecure)
if (guestToken) {
  const guestId = parseInt(guestToken, 10);
  const [guest] = await db.update(guestsTable)
    .set(updateData)
    .where(and(eq(guestsTable.id, guestId), eq(guestsTable.eventId, event.id)))
    .returning();
```

With:
```typescript
// NEW — random token lookup (SEC-04)
if (guestToken) {
  const [guest] = await db.update(guestsTable)
    .set(updateData)
    .where(and(eq(guestsTable.rsvpToken, guestToken), eq(guestsTable.eventId, event.id)))
    .returning();
```

No `parseInt`. The token is a plain string match. The `eventId` check is kept as a defense-in-depth guard even though token uniqueness already prevents cross-event hijacking.

**Fix 3 — Guest insert: ensure rsvpToken is included in response**

When guests are created via CSV import or invitation, the response should include the `rsvpToken` so callers can construct RSVP links. Search for `formatGuest` or guest insert/return patterns and ensure `rsvpToken` is in any returned guest object used to build RSVP URLs.

The RSVP link construction in the email templates (look for `guestToken` or `rsvp` in `email.ts` or campaigns) should pass `guest.rsvpToken` instead of `guest.id`. Search for `guestToken` in `artifacts/api-server/src/` and update any email link construction to use `rsvpToken`.
