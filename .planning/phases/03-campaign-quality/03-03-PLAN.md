**QUAL-03 — Structured JSON errors in backend routes**

In each route file, scan all `res.status(4xx).json(...)` and `catch` blocks. The required
error shape is: `{ error: string }`.

Common violations to find and fix:

1. `res.json({ message: "..." })` → change to `res.status(400).json({ error: "..." })`
2. `res.json({ error: err.message })` on a caught `unknown` — add type guard:
   ```ts
   const msg = err instanceof Error ? err.message : "Unexpected error";
   res.status(500).json({ error: msg });
   ```
3. Generic catch blocks that return nothing or return the raw error object.

Scope: integrations.ts, guests.ts, social.ts (the files touched in Task 1).
Do NOT re-read campaigns.ts or events.ts — those are already consistent from prior work.

Check each of these three files for:
- Any `res.json({ message: ... })` pattern (wrong key — should be `error`)
- Any bare `res.status(500).json(err)` (leaks stack trace — replace with `{ error: "Internal server error" }`)
- Any `catch (err)` block that accesses `err` without type narrowing
