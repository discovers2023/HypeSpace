# Route Protection Audit тАФ 2026-04-20

Enumerates every Express route in `artifacts/api-server/src/routes/` and classifies its authorization mechanism. The goal is to confirm that no route handling a resource-identifier path parameter leaks data across organization boundaries.

**Global middleware (from `artifacts/api-server/src/routes/index.ts`)**

1. `requireAuth` тАФ rejects with 401 if `req.session.userId` is unset. Applied to every path *except* the openPaths allowlist: `/auth/`, `/healthz`, `/health`, `/public/`, `/track/`, `/plans`, `/admin/`.
2. `requireOrgMembership` тАФ rejects with 403 if the session user is not a member of `req.params.orgId`. Applied to every `/organizations/:orgId/*` path.

**Per-route admin guard**

- `requireAdmin` тАФ enforces the admin-credential session flag. Applied per-route inside `routes/admin.ts`.

## Classification

Legend:

- ЁЯФТ `membership` тАФ covered by `requireOrgMembership` (automatic via path prefix)
- ЁЯСд `auth` тАФ covered by `requireAuth` but not org-scoped (acts on the session user, not a path param)
- ЁЯЫб `admin` тАФ covered by `requireAdmin` middleware
- ЁЯОл `token` тАФ authentication-by-opaque-random-token (no session needed)
- ЁЯМН `public` тАФ intentionally unauthenticated, product decision
- ЁЯУИ `pixel` тАФ unauthenticated tracking endpoint; response never reveals data

#### Tracking spot-check (the only ID-bearing unauthenticated routes)

Reviewed `artifacts/api-server/src/routes/tracking.ts` end-to-end (97 lines).

**`GET /track/open/:campaignId`**

- Body of every response: 1├Ч1 transparent GIF (static `PIXEL` buffer). Response length is constant regardless of campaignId validity or existence.
- Headers: `Content-Type: image/gif` + no-cache headers. No identifying headers.
- Status code: always 200. Invalid / non-existent `campaignId` тЖТ still 200 with pixel (`if (!Number.isNaN(campaignId))` gates the DB work; errors inside are swallowed into `logger.error`).
- Verdict: **No data leak.** An attacker cannot distinguish "campaign exists" from "campaign doesn't exist" by the response alone.

**`GET /track/click/:campaignId?url=...`**

- Body: empty 302 redirect.
- Redirect target: either the supplied `url` (validated against `^https?://` тАФ no `javascript:` / `data:` open redirect), or `/` as fallback.
- Status code: always 302. Invalid campaignId still redirects тАФ response does not vary.
- Verdict: **No data leak.** The redirect-target validation already closed the open-redirect concern (recorded as Phase 03 P02 decision T-03-02-02).

## Findings

- **No unprotected cross-org data paths discovered.** Every org-scoped endpoint is either covered by `requireOrgMembership` via the `/organizations/:orgId/` prefix or by `requireAdmin` (for admin-only cross-tenant operations).
- **Tracking endpoints confirmed opaque.** Both `/track/open/:campaignId` and `/track/click/:campaignId` return responses that are invariant to the validity of the campaignId, so they reveal nothing to an unauthenticated probe.
- **Public RSVP endpoints (`/public/events/:slug`) rely on slug non-enumerability.** Slug defaults use `md5(random()::text || clock_timestamp()::text)` (per Phase 1 SEC-04); event listing filters by `status = 'published'`. No change needed.
- **Invite tokens (`/invite/:token`, `/invite/:token/accept`)** use 32-byte random hex + 7-day expiry. C1 closure (quick-260420-ozg) means password overwrite is no longer possible even with a valid token.

## Residual concerns (non-blocking, already tracked in CONCERNS.md)

- **W5:** `/auth/resend-verification`, password-reset request (not yet in code), guest import, and campaign send lack rate limiters. Not an IDOR тАФ a cost / brute-force concern. Deferred.
- **W6:** dev fallback `SESSION_SECRET` is a literal string `"dev-secret-change-in-production"`. Guarded in production via the explicit throw at `app.ts:82`.

## Verdict

**C4 is closed for v1.** No code changes required beyond C1 (already shipped) and C2 + C3 (shipped in this phase). The audit stands as the reference for post-v1 reviews.
