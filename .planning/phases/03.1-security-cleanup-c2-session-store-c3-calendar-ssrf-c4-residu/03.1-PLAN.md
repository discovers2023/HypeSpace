## Task 2 — C3: SSRF guard on calendar feed import

**Files:**
- `artifacts/api-server/src/lib/ssrf-guard.ts` (new)
- `artifacts/api-server/src/routes/integrations.ts` (use guard, replace `ical.async.fromURL`)

**Action:**

1. Create `artifacts/api-server/src/lib/ssrf-guard.ts` exporting:
   ```ts
   export class UnsafeUrlError extends Error {
     constructor(public reason: string) { super(`Unsafe URL: ${reason}`); }
   }
   export async function assertSafeUrl(url: string): Promise<URL>;
   export async function safeFetchText(
     url: string,
     opts?: { maxBytes?: number; timeoutMs?: number }
   ): Promise<string>;
   ```
   - `assertSafeUrl`: parse URL; reject non-http(s); `dns.promises.lookup(hostname, { all: true })`; reject if any result is in the blocked ranges (loopback, private, link-local, IPv6 loopback/link-local/ULA).
   - `safeFetchText`: call `assertSafeUrl`; native `fetch(url, { redirect: "error", signal: AbortSignal.timeout(timeoutMs ?? 10_000) })`; stream the body and abort if `maxBytes` (default 10 MB) is exceeded; return the accumulated text.
2. In `routes/integrations.ts`:
   - Import `safeFetchText`.
   - In the function around line 491 that calls `ical.async.fromURL(calendarUrl)`, replace with:
     ```ts
     const icalText = await safeFetchText(calendarUrl, { maxBytes: 10 * 1024 * 1024 });
     const data = ical.parseICS(icalText);
     ```
   - Wrap the call in a try/catch that maps `UnsafeUrlError` → `res.status(400).json({ error: "UNSAFE_URL", message: err.reason })` at the route-handler level, and `AbortError` / size-cap → `400 "FETCH_FAILED"`.

**Verify:**
- `pnpm --filter @workspace/api-server typecheck` passes.
- Manual: attempt calendar import with `http://169.254.169.254/latest/meta-data/` → 400 UNSAFE_URL.
- Manual: attempt with `http://localhost:5432/` → 400 UNSAFE_URL.
- Manual: attempt with `http://10.0.0.1/foo.ics` → 400 UNSAFE_URL.
- Manual: attempt with a real public iCal URL (Google Calendar public feed) → succeeds, events populate.
- Manual: attempt with a file 20 MB large → 400 FETCH_FAILED (size cap hit).

**Done when:** Private-network URLs are rejected and the happy path (public feed) still works.
