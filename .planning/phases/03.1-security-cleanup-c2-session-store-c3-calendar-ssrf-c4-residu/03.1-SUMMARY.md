# Phase 03.1 — Summary

## What shipped

Three pre-ship security items from `.planning/codebase/CONCERNS.md`, each in its own atomic commit.

### C2 — Postgres-backed session store (`10b79bd`)

`express-session` now uses `connect-pg-simple` against `DATABASE_URL`:

- Sessions persist across API-server restarts.
- Multi-instance deployments can share session state.
- Production boot fails fast when `DATABASE_URL` is unset.
- Dev ergonomics preserved: `NODE_ENV!=production` + no `DATABASE_URL` → warns and falls back to `MemoryStore`.

Cookie flags (`httpOnly`, `sameSite=strict`, `secure` in prod, 7-day `maxAge`) unchanged.

**Files:** `artifacts/api-server/src/app.ts`, `artifacts/api-server/package.json`, `pnpm-lock.yaml`.
**Deps added:** `connect-pg-simple`, `pg`, `@types/connect-pg-simple`, `@types/pg`.

### C3 — SSRF guard on calendar feed import (`81d129f`)

New `artifacts/api-server/src/lib/ssrf-guard.ts`:

- `assertSafeUrl(url)` — parses, rejects non-http(s), DNS-resolves the host, rejects any result in IPv4 loopback/private/link-local/multicast/reserved ranges or IPv6 loopback/link-local/ULA (+ IPv4-mapped variants).
- `safeFetchText(url)` — calls `assertSafeUrl`, then fetches with `redirect: "error"` (closes DNS-rebinding via post-check redirect), 10-second timeout, and a streamed 10 MB body cap.
- Custom `UnsafeUrlError` exposes the rejection reason.

`routes/integrations.ts` calendar import now calls `safeFetchText` + `ical.parseICS` instead of `ical.async.fromURL`. Errors surface per-integration in the existing `errors` array returned by `GET /organizations/:orgId/calendar/events`.

**Files:** `artifacts/api-server/src/lib/ssrf-guard.ts` (new, 140 lines), `artifacts/api-server/src/routes/integrations.ts`.

### C4 — Route protection audit (`47aaeb0`)

Wrote `03.1-AUDIT.md` enumerating every Express route + its authorization mechanism across `routes/{admin,auth,campaigns,dashboard,email-provider,events,guests,health,integrations,organizations,plans,reminders,sending-domains,social,team,tracking}.ts`.

Classification taxonomy: `🔒 membership` / `👤 auth` / `🛡 admin` / `🎫 token` / `🌍 public` / `📈 pixel`.

Spot-checked `routes/tracking.ts` end-to-end — both `/track/open/:campaignId` and `/track/click/:campaignId` return responses that are invariant to `campaignId` validity (always a 1×1 GIF or a 302, constant status, constant headers, constant body). **No data leak.**

**Verdict:** no unprotected cross-org data path exists. C4 requires zero code changes beyond C1 (shipped in `quick-260420-ozg`).
