## human_verification (manual UAT before ship)

The following cannot be verified without runtime — operator must exercise before declaring v1 shipped:

1. **C2 session persistence** — with `DATABASE_URL` set and `NODE_ENV=development`:
   - Log in via the UI.
   - Restart `api-server` (`pnpm --filter @workspace/api-server dev` → Ctrl-C → relaunch).
   - Refresh the browser. **Expected:** still logged in.
   - Inspect Postgres: `SELECT COUNT(*) FROM session;` → ≥ 1.

2. **C2 production boot guard** — set `NODE_ENV=production`, unset `DATABASE_URL`, attempt to start:
   - **Expected:** process throws `DATABASE_URL env var is required in production (session store)` and exits non-zero.

3. **C2 dev fallback** — unset `DATABASE_URL`, leave `NODE_ENV` default:
   - **Expected:** server starts; log line contains `Session store: DATABASE_URL unset — using MemoryStore (dev only)` with `store: "memory"`.

4. **C3 SSRF block (cloud metadata)** — save a calendar integration with `calendarUrl = http://169.254.169.254/latest/meta-data/`, then call `GET /api/organizations/:orgId/calendar/events?year=2026&month=04`.
   - **Expected:** response is `{ events: [], errors: [{ platform: "...", error: "Unsafe URL: ...resolves to blocked address 169.254.169.254 — link-local (169.254.0.0/16)" }] }`.

5. **C3 SSRF block (localhost)** — repeat with `calendarUrl = http://127.0.0.1:5432/`.
   - **Expected:** errors array contains `Unsafe URL: ...loopback (127.0.0.0/8)`.

6. **C3 SSRF block (private)** — repeat with `calendarUrl = http://10.0.0.1/foo.ics`.
   - **Expected:** errors array contains `Unsafe URL: ...private (10.0.0.0/8)`.

7. **C3 happy path** — save a real public Google Calendar iCal feed URL. Call the same endpoint.
   - **Expected:** events list populates normally.

8. **C3 size cap** — (optional) host a 20 MB file and point the calendar at it.
   - **Expected:** errors array contains `Unsafe URL: response body exceeds 10485760 bytes`.
