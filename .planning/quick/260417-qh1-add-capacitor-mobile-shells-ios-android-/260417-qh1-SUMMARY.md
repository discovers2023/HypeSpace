# Quick Task 260417-qh1: Capacitor Mobile Shells (iOS + Android) Summary

One-liner: **Scaffolded Capacitor 6 iOS + Android wrappers around the existing React/Vite web build, threaded VITE_API_BASE_URL through the client, and opened CORS to native origins so HypeSpace can ship to the App Store and Play Store without a rewrite.**

## What Was Built

New pnpm workspace package `@workspace/hypespace-mobile` that wraps the web bundle produced by `@workspace/hypespace` into native iOS + Android shells using Capacitor 6. Plus the plumbing needed for the web bundle to talk to a remote API when running inside `capacitor://localhost`.

## Build Flow (macOS)

From the repo root, after Xcode + CocoaPods + Android Studio are installed on the Mac:

1. `pnpm --filter @workspace/hypespace run build` — builds web → `artifacts/hypespace/dist/public`
2. `pnpm --filter @workspace/hypespace-mobile run cap:sync` — copies web into native projects, runs `pod install`
3. `pnpm --filter @workspace/hypespace-mobile run cap:open:ios` — opens Xcode; hit Run
4. `pnpm --filter @workspace/hypespace-mobile run cap:open:android` — opens Android Studio; hit Run

For a hosted API, set `VITE_API_BASE_URL=https://api.hypespace.app` before step 1.

## Deferred Follow-up (REQUIRED)

**Cookie-based auth does not work inside the Capacitor shell.** The web app uses `httpOnly` + `SameSite=Strict` session cookies, which are NOT sent from `capacitor://localhost` to a remote API origin. A future quick task must:

- Add a `POST /api/auth/token` endpoint that issues a bearer token after password login.
- In the native shell, call `setAuthTokenGetter(() => nativeSecureStorage.get("token"))` from `main.tsx` when running inside Capacitor (detect via `window.Capacitor?.isNativePlatform()`).
- Store the token using `@capacitor/preferences` or Keychain/Keystore via a dedicated plugin.
- Until then, the Capacitor shells can render public pages + RSVP flows but CANNOT log in.

## Other Open Items

- **Icons + splash screens:** default Capacitor placeholders are in place (`ios/App/App/Assets.xcassets/AppIcon.appiconset/` and the various `mipmap-*` PNGs). Replace with real brand assets via `@capacitor/assets` before submitting to the App Store / Play Store.
- **No real-device testing performed** — this scaffold was produced in a Linux sandbox; the user must run `cap:sync` + `cap:open:ios` on macOS to verify the shell actually boots the web bundle.
- **Bundle identifier** (`com.discoversolutions.hypespace`) is hard-coded in `capacitor.config.ts` and baked into the native projects. If the App Store Team ID requires a different identifier, edit `capacitor.config.ts` AND update the native projects (Xcode: target → General → Bundle Identifier; Android: `android/app/build.gradle` `applicationId`).
- **Android `server.androidScheme: "https"`** is set. This means Android uses `https://localhost` as the web view origin, which is why all three Capacitor origins are in the CORS allowlist.
