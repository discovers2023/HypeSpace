Edit 2 — Add route-level guard at the top of the `Settings()` component
(currently lines 2551-2553). Update the head of the component to:

```typescript
export default function Settings() {
  const { activeOrgId, user, isLoading: authLoading } = useAuth();
  const orgId = activeOrgId;

  // Auth guard: redirect to login once auth resolves with no user.
  // Do NOT redirect while authLoading is true — that causes a flash of /login
  // for already-authenticated users on a hard refresh.
  if (!authLoading && !user) {
    return <Redirect to="/login" />;
  }
```

(Keep the rest of the function body — `useLocation`, `useSearch`, `initialTab`,
`useState`, `useEffect`, the `useGetOrganization(orgId)` call, etc. — exactly
as it is. The early return short-circuits before any of that runs only in the
logged-out case.)

Edit 3 — Add form-level disable on the Organization Profile Save button
(currently line 2723). Append `|| !orgId` to the existing `disabled` prop:

- Before: `disabled={updateOrg.isPending || !form.formState.isDirty}`
- After:  `disabled={updateOrg.isPending || !form.formState.isDirty || !orgId}`

This is belt-and-braces: the route guard above means an unauthenticated user
will not reach the form, but if `/auth/me` is still in flight (`authLoading
=== true`, `user === null`, `activeOrgId === 0`) the form may briefly render
on hard refresh — disabling Save in that window prevents the `PUT
/organizations/0` regression.

Do NOT add the `|| !orgId` guard symmetrically to the Branding (line 2086)
or AI Settings (line 2224) Save buttons. They live in separate sub-components
that the route-level guard already prevents from mounting in the logged-out
case, and per scope: "add the guard symmetrically only if trivially
