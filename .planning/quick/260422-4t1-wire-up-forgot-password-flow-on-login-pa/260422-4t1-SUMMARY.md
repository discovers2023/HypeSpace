# Phase quick-260422-4t1 Plan 01: Forgot Password Flow Summary

**One-liner:** End-to-end self-service password reset via hashed single-use tokens, enumeration-safe API, and matching two-column pages mirroring login.tsx.

## What Was Built

### Task 1: Schema + Backend

- **`lib/db/src/schema/password_reset_tokens.ts`** тАФ New Drizzle table with `user_id` FK, `token_hash` (SHA-256 hex, unique), `expires_at` (1h), `used_at` (null until consumed), `created_at`. Pushed to Postgres via `pnpm db:push`.

- **`sendPasswordResetEmail()`** in `email.ts` тАФ Mirrors `sendVerificationEmail` structure: gradient header, button CTA, copy-paste fallback link, 1-hour expiry notice. Calls the generic `sendEmail()` helper (Ethereal fallback in dev).

- **`POST /auth/forgot-password`** тАФ Always returns `{ message: "If an account exists..." }` (200) regardless of whether email exists or token generation fails. Rate-limited by existing `authLimiter`. `hashToken()` helper uses `crypto.createHash("sha256")`.

- **`POST /auth/reset-password`** тАФ Validates token hash + `expires_at > now` + `used_at IS NULL`. On valid token: bcrypt hashes new password, updates `users.password_hash`, sets `password_reset_tokens.used_at`. Returns 400 `INVALID_TOKEN` for expired/used/unknown tokens.

### Task 2: Frontend Pages

- **`/forgot-password`** тАФ Email input тЖТ form submits тЖТ regardless of response swaps to confirmation state ("Check your email" with submitted email interpolated). Enumeration-safe: confirmation always shows. Footer link back to /login.

- **`/reset-password`** тАФ Reads `token` from `window.location.search`. No token тЖТ static error state with link to /forgot-password. Valid token present тЖТ two password fields with show/hide eye toggles + Zod `.refine()` confirm match. On success: `useToast()` notification + `setLocation("/login")`. On API error: inline destructive banner with server message.

- **`App.tsx`** тАФ Both routes registered as public (outside `RequireAuth`) alongside `/login`, `/register`, `/accept-invite`.

- **`login.tsx` line 152** тАФ Changed `href="#"` to `href="/forgot-password"`. Only change to this file.
