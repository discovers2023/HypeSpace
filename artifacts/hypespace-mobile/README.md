# @workspace/hypespace-mobile

Capacitor 6 shell that wraps the HypeSpace web build for native iOS and Android
distribution (App Store + Google Play). The web bundle is the same one served
by the browser — this package only adds the native project shells.

## Identity

- **App ID:** `com.discoversolutions.hypespace`
- **App Name:** `HypeSpace`
- **webDir:** `../hypespace/dist/public` (produced by `pnpm --filter @workspace/hypespace run build`)
- **Capacitor:** 6.x (`@capacitor/core`, `@capacitor/cli`, `@capacitor/ios`, `@capacitor/android`)

## Build + Open Flow (macOS)

Xcode and Android Studio are required on your Mac. iOS also needs CocoaPods
installed (`sudo gem install cocoapods` or `brew install cocoapods`).

```bash
# 1. Build the web bundle (from repo root)
pnpm --filter @workspace/hypespace run build

# 2. Sync the web assets into the native projects + install iOS Pods
pnpm --filter @workspace/hypespace-mobile run cap:sync

# 3a. iOS — opens Xcode, where you hit Run to deploy to a simulator / device
pnpm --filter @workspace/hypespace-mobile run cap:open:ios

# 3b. Android — opens Android Studio, then Run on a simulator / device
pnpm --filter @workspace/hypespace-mobile run cap:open:android
```

Shortcut: `pnpm --filter @workspace/hypespace-mobile run build:mobile` runs
steps 1 + 2 together.

## API Endpoint Configuration

The web bundle is baked into the native app at build time. In browser
deployments, API calls use the relative path `/api/...` (Vite dev proxy or
same-origin reverse proxy handles routing). Inside a Capacitor shell there is
no proxy — the web view runs at `capacitor://localhost` and has to be told
where the API server lives.

**Set `VITE_API_BASE_URL` BEFORE running `pnpm build`:**

```bash
# Hosted API (production native build)
VITE_API_BASE_URL=https://api.hypespace.app \
  pnpm --filter @workspace/hypespace run build

# LAN API (testing a simulator against a local dev server)
VITE_API_BASE_URL=http://192.168.1.42:4000 \
  pnpm --filter @workspace/hypespace run build
```

When `VITE_API_BASE_URL` is unset or empty, the web client falls back to
relative `/api/...` — this is the right default for browser deployments and
must not change.

The API server's CORS allowlist already includes the Capacitor native origins
(`capacitor://localhost`, `http://localhost`, `https://localhost`) in
development. For production, set `ALLOWED_ORIGINS` on the API server to include
your hosted origin AND the Capacitor origins, comma-separated. Example:

```bash
ALLOWED_ORIGINS=https://app.hypespace.app,capacitor://localhost,https://localhost
```

## Icons and Splash Screens

Default Capacitor placeholders are used at initial scaffold time. Before
shipping, generate real assets with `@capacitor/assets`:

```bash
pnpm --filter @workspace/hypespace-mobile add -D @capacitor/assets
# Drop a 1024x1024 icon.png and 2732x2732 splash.png into assets/
pnpm --filter @workspace/hypespace-mobile exec capacitor-assets generate
```

## Known Limitation: Cookie Auth Does Not Work Inside Capacitor

The HypeSpace web app uses `httpOnly` + `SameSite=Strict` session cookies.
These cookies are NOT sent from `capacitor://localhost` to a remote API
origin, so **login will not work inside the Capacitor shell in this v1**.

A follow-up task will migrate to bearer-token auth:

1. Add `POST /api/auth/token` that issues a bearer token after password login.
2. In `main.tsx`, detect `window.Capacitor?.isNativePlatform()` and call
   `setAuthTokenGetter(() => nativeSecureStorage.get("token"))`.
3. Store the token via `@capacitor/preferences` (or Keychain/Keystore through
   a dedicated plugin).

Until then, the native shell can render public pages and public RSVP flows
but cannot reach authenticated routes.

## What Lives Where

```
artifacts/hypespace-mobile/
├── package.json           # pnpm workspace manifest + cap:* scripts
├── capacitor.config.ts    # appId, appName, webDir, androidScheme
├── .gitignore             # top-level ignores (node_modules, Pods, build, etc.)
├── README.md              # this file
├── ios/                   # Xcode workspace (generated by `cap add ios`)
│   └── App/App.xcworkspace/
└── android/               # Gradle project (generated by `cap add android`)
    └── build.gradle
```

Do not commit files under `ios/App/Pods/`, `ios/DerivedData/`,
`android/build/`, `android/app/build/`, or `android/.gradle/` — the generated
`.gitignore` files handle that.
